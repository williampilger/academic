### 1. Explique por que um sistema de software usado em um ambiente real deve mudar ou torna-se progressivamente menos útil.
```
As necessidades do ambiente e dos usuários muda com o passar do tempo.
Portanto, não adaptar o sistema às novas demandas do mercado, significa a sua descontinuação.
```

### 2. O que é degradação de software e o que a acarreta?
```
A degradação do software ocorre com o passar do tempo, com a ferramenta deixando de atender a
algum requisito da atualidade, do novo cenário. Ou seja, houve uma nova necessidade, um
novo requisito, mas a funcionalidade ainda não foi implementada. A Experiência do usuário
e/ou a utilidade do sistema foi degradada, e só será reconstituida se o software for 
atualizado ou substituido.
```

### 3. Explique a diferença entre Reengenharia e Refatoração.
```
A Reengenharia envolve a reestruturação completa de um sistema/software.
Enquanto a refatoração consiste na atualização de código com o intuito de melhorar aspectos
como a legibilidade ou eficiência do código, mas sem alterar os requisitos funcionais.
```

### 4. O que é Engenharia Reversa?
```
O processo de obter um código que entrega o mesmo resultado de uma aplicação da qual
não se possui o código-fonte. Poder-se-ia chamar isso de cópia/pirataria, dependendo
do contexto.
```

### 9. Quais são os principais fatores que afetam os custos da reengenharia de sistemas?
```
Como trata-se de uma manutenção, a atividade por si só já adiciona tarefas e complexidade quando
comparada a outras atividades de desenvolvimento de software. Como por exemplo, a necessidade de
entender o sistema existente, ou estudar os efeitos colaterais das alterações em outras funcionalidades
do ecosistema existente.
```

### 10. Em quais circunstâncias uma organização pode decidir descartar um sistema?

de um sistema novo, que atenda aos requisitos da atualidade.
